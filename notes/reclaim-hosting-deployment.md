# Reclaim Hosting Deployment

## Purpose

Reclaim Hosting is commonly used in academic and digital humanities contexts for web hosting, CMS projects, and digital publishing. An Omeka S archive deployment involves both technical setup and practical client support.

## High-Level Workflow

1. Confirm the domain, hosting environment, and project needs.
2. Install or configure Omeka S in the hosting environment.
3. Set up site structure, modules, themes, and public pages.
4. Configure archive resource templates, metadata, and item organization.
5. Add or import initial content.
6. Review public pages, search, navigation, and responsive layouts.
7. Test basic admin workflows for future maintainers.
8. Handoff documentation and support guidance.

## Infrastructure Concepts

- Domain: the public address users visit.
- DNS: the system that points the domain to hosting infrastructure.
- Hosting: the server environment where Omeka S runs.
- CMS: the content management system used to manage public pages and records.
- Database: stores records, configuration, and content relationships.
- Theme: controls much of the public-facing presentation.

## Practical Support Tasks

- Coordinating access and permissions.
- Explaining admin workflows.
- Testing public pages and record views.
- Troubleshooting upload, layout, or navigation issues.
- Supporting future expansion planning.
- Documenting repeatable steps for client ownership.

## What This Demonstrates

- Practical hosting and deployment literacy.
- CMS implementation support.
- User/client training.
- Technical documentation.
- Platform support across domain, hosting, CMS, content, and public UX layers.
